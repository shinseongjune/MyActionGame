# MyActionGame 다음 채팅 핸드오프

이 문서는 다음 채팅에서 바로 이어 작업하기 위한 현재 상태와 우선순위 메모다.

## 현재 체크포인트

- 브랜치: `main`
- 최신 푸시 커밋: `0a6f9b2 Add player attack and parry checkpoint`
- 직전 기준 빌드: `MyActionGame Win64 Development` 성공
- 첫 플레이어블 루프는 이미 `c31b332 Complete first playable loop`에서 검증됨
- 플레이어 공격과 패리는 구현된 상태로 본다
- 적 공격을 `WeaponTraceComponent`로 바꾸는 작업은 의도적으로 보류한다

관련 문서:

- `Docs/FirstStage_BlockoutPlan.md`

## 이미 된 것

- `EncounterStateComponent`: 전투 구역 진입, 적 등록, 클리어 이벤트 관리
- `StageBlockoutBuilder`: 첫 스테이지 블록아웃 생성 도구
- 첫 스테이지 흐름: `입장 -> 전투 -> 클리어 -> 진행 -> 두 번째 전투 -> 보상/종료`
- `PlayerFailureComponent`: 플레이어 HP 계산과 분리된 사망/리셋 이벤트 처리
- `WeaponTraceComponent` / `WeaponTraceAnimNotifyState`: 플레이어 무기 소켓 기반 공격 판정
- 플레이어 패리 기본 흐름
- Sevarog 플레이어 애니메이션/몽타주를 `_Game` 쪽으로 정리

## 백로그

### 플레이어

- 콤보 2타/3타 연결
- 스킬 입력과 실제 효과 연결
- 파괴 가능한 오브젝트 등 상호작용 액션
- 넉백, 스턴, 사망, 피격 리액션 처리
- 패리 성공/실패 피드백 강화
- 공격, 패리, 회피, 스킬 사이 입력 우선순위 정리
- 카메라 충돌, 시야 가림, 락온 여부 검토

### 적

- 팔/다리 등 타격 위치별 반응 추가
- 정예 적 추가
- 보스 또는 미니보스 추가
- 공격 전 텔레그래프 강화
- 피격, 스턴, 처형 가능, 사망 상태의 규칙 정리
- 다수 적이 동시에 붙을 때 전투 가독성 조정
- 적 공격 WeaponTrace 전환은 지금 당장 하지 않는다

### 스테이지

- `BP_BreakableObstruction` 같은 파괴 가능한 오브젝트 추가
- 파괴 오브젝트를 길막이, 보상, 전투 리듬에 연결
- 맵 꾸미기: 바닥, 벽, 조명, 엄폐물, 시선 유도
- 승리 조건과 패배 조건 명확화
- 보상/종료 지점의 시각적 표시
- 체크포인트 또는 리스타트 위치 기준 정리
- 카메라가 망가지지 않는 방 크기와 통로 폭 조정

### UI

- 플레이어 체력 표시
- Aegis 표시
- 상처/Wound 표시
- 적 또는 보스 체력바
- 현재 가능한 키 입력 표시
- 패리 가능 타이밍, 스킬 쿨다운, 상호작용 가능 표시
- 승리 HUD
- 패배 HUD
- 리스타트/나가기 버튼
- 임시 `Print String` 의존 제거

### 시스템

- 클리어 처리
- 리스타트 처리
- 실패 후 입력 잠금과 복구
- 사운드 처리
- 이펙트 처리
- 옵션: 키 입력 변경
- 옵션: 마스터/효과음/음악 볼륨 조절
- 저장까지는 아직 필수 아님. 먼저 한 판 리스타트 루프를 안정화한다
- 디버그용 전투 로그 또는 화면 표시 정리

### 추가로 빠지기 쉬운 항목

- 전투 카메라와 락온 정책
- 히트스톱, 카메라 셰이크, 피격 사운드 같은 타격감 피드백
- 보상 구조: 클리어하면 무엇을 얻는지
- 튜토리얼성 안내: 첫 공격, 첫 패리, 첫 상호작용
- 품질 기준: 한 판이 깨지는 대표 상황을 체크리스트화
- 에셋 정리 규칙: `_Game` 아래에 실제 프로젝트용 에셋을 모으고, 외부 팩 원본은 가능하면 직접 수정하지 않는다

## 추천 작업 순서

1. `BP_BreakableObstruction`을 실제 레벨 진행에 연결한다.
2. 승리/패배 HUD와 리스타트 버튼을 붙인다.
3. 플레이어 HP/Aegis/Wound UI를 만든다.
4. 패리 성공/실패 피드백을 시각/사운드로 강화한다.
5. 콤보 2타/3타를 붙인다.
6. Room 02를 정예 적 또는 미니보스 테스트 공간으로 바꾼다.
7. 보스 체력바와 승리 연출을 붙인다.
8. 옵션 화면은 전투 루프가 한 번 더 안정된 뒤 붙인다.
